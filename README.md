## <u>Teamate</u> Project Background

### 필요성

![클라우드 vs 엣지 컴퓨팅](002%20Presentation/image/image.png)

- **실시간 처리 요구**: 낙상 감지는 즉각적인 대응이 필요한 응급 상황으로, 클라우드 기반 처리의 네트워크 지연(latency)은 치명적
- **개인정보 보호**: 엣지 디바이스에서 직접 처리하여 민감한 건강 데이터를 외부 서버로 전송하지 않음
- **네트워크 독립성**: 인터넷 연결이 불안정한 환경에서도 안정적으로 작동

### 기존 해결책의 문제점

- **웨어러블 기기의 한계**: 착용 불편, 충전 필요, 착용 거부감
- **비전 기반 시스템**: 프라이버시 침해 우려, 조명 조건에 민감
- **클라우드 의존성**: 네트워크 지연으로 인한 실시간 대응 지연, 통신 비용 발생

### 데이터 수집 및 전처리

![센서 데이터 실시간 수집](002%20Presentation/image/rasp.gif)

![걷기 vs 뛰기 센서 데이터 비교](002%20Presentation/image/data.png)

- **샘플링 주파수**: 50Hz로 IMU 데이터 실시간 수집
- **데이터 정규화**: Min-Max Scaling을 통한 센서 데이터 정규화
- **특징 추출**: 시간 윈도우 기반 통계적 특징 추출(평균, 분산, 최대/최소값)

### 이상 탐지 모델

![Autoencoder 모델 구조](002%20Presentation/image/ae.png)

**Autoencoder 기반 이상 탐지**
- **학습 데이터**: 정상 행동 패턴(걷기, 앉기, 서기)만 학습
- **이상 감지 원리**: 재구성 오차(Reconstruction Error)가 임계값을 초과하면 낙상으로 판단
- **장점**: 비정상 데이터 없이도 학습 가능, 새로운 낙상 패턴에 대한 일반화 능력 우수

### 전체 시스템 아키텍처

![시스템 구조도](002%20Presentation/image/system.png)

1. **센서 계층**: 아두이노 + IMU 센서에서 가속도/각속도 데이터 수집
2. **엣지 처리 계층**: 라즈베리파이에서 실시간 데이터 전처리 및 모델 추론
3. **알림 계층**: 낙상 감지 시 즉각적인 알림 발송


## Conclusion

### 실시간 낙상 감지

![낙상 감지 시스템 시연](002%20Presentation/image/test.gif)

**성능 지표**
- **정확도(Accuracy)**: 95.2%
- **재현율(Recall)**: 93.8% (실제 낙상을 정확히 감지)
- **정밀도(Precision)**: 94.5% (오탐지 최소화)
- **추론 시간**: 평균 15ms (실시간 처리 가능)


![이상 탐지 Foundation Model로의 확장](002%20Presentation/image/conclusion.png)

### 확장 가능성

1. **다양한 이상 행동 감지**: 낙상 외 실신, 발작 등 다양한 응급 상황 감지
2. **Foundation Model 구축**: 대규모 센서 데이터로 사전 학습된 범용 이상 탐지 모델 개발
3. **멀티 모달 융합**: 카메라, 마이크 등 다중 센서 데이터 통합 분석
4. **개인화 모델**: 사용자별 행동 패턴 학습을 통한 맞춤형 이상 탐지
